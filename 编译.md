# Adafruit_nRF52_Bootloader 编译.md

本文记录从拉取 `Adafruit_nRF52_Bootloader` 仓库，到编译 `pca10056` 或自定义 `snail60` bootloader 成功的完整流程。

适用场景：

- Windows CMD 环境
- nRF52840 芯片
- Adafruit nRF52 Bootloader
- ZMK 键盘固件
- 需要生成 `.hex` / `.uf2` 文件

---

## 1. 准备工具

需要提前安装：

| 工具 | 作用 |
|---|---|
| Git | 拉取仓库和子模块 |
| make | 执行 Makefile 编译 |
| Arm GNU Toolchain | 编译 ARM Cortex-M 程序 |
| nRF Command Line Tools | 使用 `nrfjprog` 烧录 |
| Python | 执行脚本和安装依赖 |
| intelhex | 生成 / 合并 hex 文件 |

---

## 2. 检查环境

在 CMD 中执行下面命令：

```cmd
REM 检查 ARM GCC 是否正常安装
arm-none-eabi-gcc --version

REM 检查 nrfjprog 是否正常安装
nrfjprog --version

REM 安装 intelhex，用于处理 hex 文件
python3 -m pip install intelhex

REM 检查 make 是否可用
make --version
```

如果 `make --version` 提示：

```text
'make' 不是内部或外部命令，也不是可运行的程序
```

说明 Windows CMD 没有找到 `make`。可以使用下面任意一种方式解决：

1. 使用 Git Bash 编译；
2. 安装 MSYS2，并安装 `make`；
3. 把已有的 `make.exe` 所在目录加入系统环境变量 `Path`。

## 3. 拉取仓库

如果还没有仓库，执行：

```cmd
REM 进入 D 盘
D:

REM 进入 github 工作目录
cd \github

REM 拉取 Adafruit nRF52 Bootloader 仓库，并同时拉取子模块
git clone --recurse-submodules https://github.com/adafruit/Adafruit_nRF52_Bootloader.git

REM 进入 bootloader 工程目录
cd Adafruit_nRF52_Bootloader
```

如果已经拉取过仓库，只需要进入目录：

```cmd
REM 进入 bootloader 工程目录
cd /d D:\github\Adafruit_nRF52_Bootloader
```

## 4. 拉取 / 更新子模块

第一次编译前必须执行：

```cmd
REM 初始化并更新全部子模块
git submodule update --init --recursive
```

如果看到很多类似下面的信息，说明子模块正在正常拉取：

```text
Submodule path 'lib/tinyusb': checked out 'xxxxxxx'
Submodule path 'lib/uf2': checked out 'xxxxxxx'
```

最后重新回到命令行提示符，表示执行完成：

```text
D:\github\Adafruit_nRF52_Bootloader>
```

## 5. 编译官方 pca10056

`pca10056` 对应 Nordic nRF52840 DK。

执行：

```cmd
REM 清理 pca10056 旧编译文件
make BOARD=pca10056 clean

REM clean 后可能会删除 _build 目录，所以这里重新创建 _build 父目录
if not exist _build mkdir _build

REM 编译 pca10056 bootloader
make BOARD=pca10056 all
```

如果出现下面错误：

```text
mkdir: can't create directory '_build/build-pca10056': No such file or directory
make: *** [Makefile:472: _build/build-pca10056] Error 1
```

说明 `_build` 父目录不存在。执行：

```cmd
REM 手动创建 _build 父目录
if not exist _build mkdir _build

REM 重新编译 pca10056
make BOARD=pca10056 all
```

## 6. 创建自己的板子 snail60

如果要配置自己的 bootloader，不建议直接改 `pca10056`，建议复制一份新的板级目录。

执行：

```cmd
REM 复制 pca10056 板级配置，创建自己的 snail60 板级目录
xcopy /E /I src\boards\pca10056 src\boards\snail60
```

创建后目录应为：

```text
Adafruit_nRF52_Bootloader
└── src
    └── boards
        └── snail60
            ├── board.h
            └── board.mk
```

## 8. 修改 snail60 的 board.h

打开：

```text
src\boards\snail60\board.h
```

可以按下面模板修改：

```c
#ifndef SNAIL60_H                                      // 防止头文件被重复包含
#define SNAIL60_H                                      // 定义 SNAIL60_H 头文件保护宏

#define LEDS_NUMBER       1                            // bootloader 使用的 LED 数量
#define LED_PRIMARY_PIN   15                           // 主 LED 引脚，示例为 P0.15
#define LED_SECONDARY_PIN 15                           // 没有第二个 LED 时，可以先和主 LED 写一样
#define LED_STATE_ON      0                            // LED 点亮电平，0 表示低电平点亮，1 表示高电平点亮

#define BUTTON_DFU        11                           // USB UF2 bootloader 进入按键，没有按键时填一个未使用 IO
#define BUTTON_DFU_OTA    12                           // BLE OTA bootloader 进入按键，没有按键时填一个未使用 IO
#define BUTTON_PULL       NRF_GPIO_PIN_PULLUP          // 按键内部上拉，空闲 IO 默认保持高电平

#define BLEDIS_MANUFACTURER "Snail"                    // BLE OTA 设备信息中的厂商名
#define BLEDIS_MODEL        "Snail60"                  // BLE OTA 设备信息中的型号名

#define USB_DESC_VID          0x239A                   // USB VID，个人测试可以先沿用 Adafruit 的 VID
#define USB_DESC_UF2_PID      0x00DA                   // UF2 模式 USB PID
#define USB_DESC_CDC_ONLY_PID 0x00DA                   // CDC 串口模式 USB PID

#define UF2_VOLUME_LABEL      "SNAIL60   "             // UF2 U盘卷标，建议固定 11 个字符，不足用空格补齐
#define UF2_PRODUCT_NAME      "Snail60 nRF52840"       // INFO_UF2.TXT 中显示的产品名
#define UF2_BOARD_ID          "nRF52840-snail60-v1"    // INFO_UF2.TXT 中显示的板子 ID
#define UF2_INDEX_URL         "https://example.com"    // INFO_UF2.TXT 中显示的网址

#endif                                                 // 结束头文件保护
```

注意：

- `UF2_VOLUME_LABEL` 是电脑上显示的 U 盘名字；
- `UF2_PRODUCT_NAME` 是 `INFO_UF2.TXT` 里面显示的产品名；
- `UF2_VOLUME_LABEL` 建议保持 11 个字符；
- 不建议使用中文卷标；
- 没有实体按键时，`BUTTON_DFU` 和 `BUTTON_DFU_OTA` 不建议删除，填未使用 IO 即可；
- 双击 Reset 进入 bootloader 不依赖 `BUTTON_DFU`；
- ZMK 的 `&bootloader` 也不依赖实体 DFU 按键。

## 9. 是否需要配置 DCDC / REGOUT0

如果你的硬件是外部 LDO 供电：

```text
USB 5V / 电池 → 外部 LDO → 3.3V → nRF52840 VDD
```

一般不需要配置：

```c
#define UICR_REGOUT0_VALUE ...                         // 外部 LDO 供电时通常不需要配置 REGOUT0
#define ENABLE_DCDC_0 1                                // 没有对应 DCDC 电感时不要乱开
#define ENABLE_DCDC_1 1                                // 没有对应 DCDC 电感时不要乱开
```

如果你的硬件是 VDDH 供电：

```text
USB 5V / 电池 → nRF52840 VDDH
nRF52840 VDD → 给芯片和外设供电
```

可以根据硬件考虑：

```c
#define UICR_REGOUT0_VALUE UICR_REGOUT0_VOUT_3V3       // 设置 VDDH 模式下 VDD 输出为 3.3V
#define ENABLE_DCDC_0 1                                // 使能 REG0 DCDC，前提是硬件有对应电感
#define ENABLE_DCDC_1 1                                // 使能 REG1 DCDC，前提是硬件有对应电感
```

重点：

- 设置输出电压靠 `UICR_REGOUT0_VALUE`；
- `ENABLE_DCDC_0` / `ENABLE_DCDC_1` 是切换稳压方式；
- 没有 DCDC 电感时，不要强行打开 DCDC。

## 12. 关于 adafruit-nrfutil 报错

如果最后出现：

```text
process_begin: CreateProcess(NULL, adafruit-nrfutil dfu genpkg ...)
make (e=2): 系统找不到指定的文件。
make: *** [Makefile:522: xxx.zip] Error 2
```

这不代表 `.hex` 编译失败。

它只是说明缺少：

```text
adafruit-nrfutil
```

这个工具只影响 `.zip` DFU 包生成。

如果你只需要直接烧录 `.hex`，可以不管这个错误。

如果你需要生成 `.zip`，再安装：

```cmd
REM 安装 adafruit-nrfutil，用于生成 DFU zip 包
python -m pip install --user adafruit-nrfutil
```

如果安装后仍然找不到：

```cmd
REM 查看 Python 用户目录
python -m site --user-base

REM 临时把 Python Scripts 目录加入当前 CMD 的 PATH
for /f "delims=" %i in ('python -m site --user-base') do set PATH=%i\Scripts;%PATH%

REM 检查 adafruit-nrfutil 是否能被找到
where adafruit-nrfutil
```

## 13. ZMK 推荐烧录哪个 hex

如果你是 ZMK 键盘，通常优先烧不带 S140 的 bootloader：

```text
snail60_bootloader-xxxxxxx.hex
```

不优先烧：

```text
snail60_bootloader-xxxxxxx_s140_6.1.1.hex
```

原因：

- ZMK 基于 Zephyr；
- ZMK 不依赖传统 nRF5 SDK 的 S140 SoftDevice；
- 不带 S140 更干净；
- 后续如果使用 `nrf52840-nosd` 布局，可以释放更多 Flash / Storage 空间。

如果你当前 ZMK 固件是普通带 SoftDevice 预留布局，也可以使用带 S140 的合并 hex，但对 ZMK 蓝牙本身没有额外帮助。

## 14. 烧录 snail60 bootloader

先查看生成的真实文件名：

```cmd
REM 查看 snail60 编译输出文件
dir _build\build-snail60
```

假设生成文件为：

```text
snail60_bootloader-c67f0bc.hex
```

烧录命令：

```cmd
REM 擦除整片 nRF52840
nrfjprog -f nrf52 --eraseall

REM 烧录 snail60 bootloader，文件名按实际生成结果修改
nrfjprog -f nrf52 --program _build/build-snail60/snail60_bootloader-c67f0bc.hex --verify

REM 复位芯片
nrfjprog -f nrf52 --reset
```

如果电脑连接了多个 J-Link，可以先查看序列号：

```cmd
REM 查看已连接的 J-Link 设备
nrfjprog --ids
```

指定序列号烧录：

```cmd
REM 擦除指定 J-Link 对应的 nRF52840
nrfjprog -f nrf52 --snr 123456789 --eraseall

REM 烧录指定 J-Link 对应的 nRF52840
nrfjprog -f nrf52 --snr 123456789 --program _build/build-snail60/snail60_bootloader-c67f0bc.hex --verify

REM 复位指定 J-Link 对应的 nRF52840
nrfjprog -f nrf52 --snr 123456789 --reset
```

## 15. 测试是否成功

烧录完成后，可以用下面方式进入 bootloader：

1. 双击 Reset；
2. ZMK 中使用 `&bootloader`；
3. 如果有 DFU 按键，按住 DFU 再复位。

进入成功后，电脑会出现一个 U 盘。

如果你配置了：

```c
#define UF2_VOLUME_LABEL "SNAIL60   "             // UF2 U盘卷标
```

电脑上应该显示类似：

```text
SNAIL60
```

然后可以把 ZMK 生成的 `.uf2` 文件拖进去。

## 16. 常见问题

### 16.1 编译时报 `_build/build-snail60` 无法创建

原因：

```text
_build 父目录不存在
```

解决：

```cmd
REM 创建 _build 父目录
if not exist _build mkdir _build

REM 重新编译
make BOARD=snail60 all
```

### 16.2 编译时报找不到 adafruit-nrfutil

原因：

```text
缺少 adafruit-nrfutil，无法生成 zip 包
```

解决：

```cmd
REM 安装 adafruit-nrfutil
python -m pip install --user adafruit-nrfutil
```

如果只烧 `.hex`，可以忽略这个错误。

### 16.3 双击 Reset 能进 bootloader 吗

可以。

双击 Reset 进入 bootloader 不依赖实体 DFU 按键。

### 16.4 没有按键可以不写 BUTTON_DFU 吗

不建议删除。

建议保留：

```c
#define BUTTON_DFU        11                           // 没有实体按键时，填一个未使用 IO
#define BUTTON_DFU_OTA    12                           // 没有实体按键时，填一个未使用 IO
#define BUTTON_PULL       NRF_GPIO_PIN_PULLUP          // 内部上拉，避免空脚乱跳
```

### 16.5 NRF52BOOT 可以改名字吗

可以。

在自己的 `board.h` 中加入：

```c
#define UF2_VOLUME_LABEL "SNAIL60   "                  // UF2 U盘卷标，最多建议 11 个字符
```

## 17. 最常用的一套命令

如果已经创建好 `snail60`，日常重新编译使用下面命令：

```cmd
REM 进入 bootloader 工程目录
cd /d D:\github\Adafruit_nRF52_Bootloader

REM 清理 snail60 旧编译文件
make BOARD=snail60 clean

REM 重新创建 _build 父目录
if not exist _build mkdir _build

REM 编译 snail60 bootloader
make BOARD=snail60 all

REM 查看编译输出文件
dir _build\build-snail60
```

烧录：

```cmd
REM 擦除整片 nRF52840
nrfjprog -f nrf52 --eraseall

REM 烧录 snail60 bootloader，注意文件名要改成实际生成的名字
nrfjprog -f nrf52 --program _build/build-snail60/snail60_bootloader-c67f0bc.hex --verify

REM 复位芯片
nrfjprog -f nrf52 --reset
```

## 18. 一句话总结

从零开始的完整流程是：

```text
安装工具 → 拉取仓库 → 更新子模块 → 复制 pca10056 创建 snail60 → 修改 board.h / board.mk → make BOARD=snail60 all → 烧录 snail60_bootloader-xxxxxxx.hex → 双击 Reset 测试 UF2 U盘
```
